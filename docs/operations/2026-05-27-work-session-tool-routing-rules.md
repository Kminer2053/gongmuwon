# 업무대화 연계도구 라우팅 규칙 정리

작성일: 2026-05-27

## 목적

업무대화는 일반 LLM 채팅이면서 동시에 공무원 앱의 일정, 파일찾기, 지식폴더, 문서작성 기능을 호출하는 중심 화면이다. 따라서 사용자가 자연어로 업무를 지시했을 때 무조건 도구를 실행하면 안 되고, 확실한 도구 후보를 찾은 뒤 대화창 안에서 먼저 확인을 받아야 한다.

이번 기준은 우측 승인 패널을 재사용하지 않는다. 사용자가 작업 흐름을 끊기지 않도록 업무대화 안에서 `도구 실행 확인 -> 동의/거절 -> 입력값 정리 -> 도구 실행` 순서로 처리한다.

## 현재 라우팅 단계

1. 사용자가 업무대화에 메시지를 입력한다.
2. 기존에 대기 중인 도구 확인 요청이 있으면, 현재 메시지를 동의/거절 응답으로 먼저 해석한다.
3. 사용법 질문이면 로컬 사용법 안내를 바로 반환한다.
4. 초고확신 규칙 라우터가 도구 후보를 판정한다.
5. 일정 등록, 문서작성, 지식폴더 검색, 파일찾기는 즉시 실행하지 않고 확인 메시지를 먼저 반환한다.
6. 사용자가 동의하면 LLM을 최소 추론 모드로 호출해 원문에서 도구 입력값만 JSON으로 추출한다.
7. 추출된 입력값으로 실제 도구를 호출한다.
8. 실행 결과에는 내부 룰 ID가 아니라 `이렇게 이해했어요`와 사용자 친화적인 도구명을 표시한다.

## 도구별 규칙

| 도구 | 확인 메시지 | LLM 추출값 | 실행 |
| --- | --- | --- | --- |
| 일정 등록 | `일정 등록으로 처리할까요?` | 제목, 시작/종료 시간, 장소, 설명 | 업무일정 캘린더에 일정 생성 |
| 문서작성 | `문서작성으로 처리할까요?` | 제목, 출력유형, 목적, 작성지시, 대상, 분량 등 | 업무대화/연결파일/지식근거 기반 HWPX 생성 |
| 지식폴더 검색 | `지식폴더 검색으로 처리할까요?` | 핵심 검색어, 검색 개수 | GraphRAG 검색 및 출처 링크 반환 |
| 파일찾기 | `파일찾기로 처리할까요?` | 핵심 파일 검색어, 검색 개수 | 로컬 파일명 인덱스와 지식폴더 파일 검색 |

## 일반 대화로 남겨야 하는 경우

아래 요청은 도구 실행이 아니라 일반 답변으로 처리한다.

- `문서작성에는 뭐가 중요할까?`
- `파일찾기 사용법 알려줘`
- `일정 기능 설명해줘`
- `보고서 잘 쓰는 법 알려줘`

특히 `문서작성`이라는 단어가 포함되어도, `중요할까`, `방법`, `요령`, `팁`, `설명`처럼 조언을 구하는 표현이면 HWPX 파일을 만들지 않는다.

## 피드백 루프

도구 확인 요청은 `work_session_tool_confirmations` 테이블에 기록한다.

- `pending`: 사용자 확인 대기
- `accepted`: 사용자가 동의해 도구 실행
- `rejected`: 사용자가 취소 또는 부정
- `superseded`: 새 확인 요청으로 대체

이 구조는 이후 “도구 판단이 맞았는지”를 점수화하는 출발점이다. 현재는 실행 전 확인과 기록까지 반영했고, 향후 100건 이상 누적 시 사용자 동의/거절 패턴을 분석해 규칙 후보를 개선할 수 있다.

## 문서작성 컨텍스트 정리

문서작성은 업무대화 전체를 그대로 복사하지 않는다. 대화세션, 연결 일정, 연결 파일, Reference Set, GraphRAG 근거를 모아 WorkSession Brief와 Document Plan을 만든 뒤 HWPX에 반영한다.

도구 확인용 메시지(`...으로 처리할까요?`)는 업무 원천 맥락이 아니므로 문서작성 컨텍스트에서 제외한다. 이렇게 해야 최종 보고서에 라우팅 안내문이나 확인 요청 문장이 섞이지 않는다.

## UI 표시 원칙

- 내부 action id(`document.create`, `file.search`)는 사용자에게 노출하지 않는다.
- 업무대화 출처 라벨은 `공무 도구 / 문서작성 확인`처럼 기능명 중심으로 표시한다.
- 우측 승인 패널은 이 흐름에 사용하지 않는다.
- 파일찾기 결과는 파일명, 일치 이유, 파일 열기 경로, 폴더 열기 경로를 함께 표시한다.
