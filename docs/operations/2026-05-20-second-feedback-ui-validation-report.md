# 2026-05-20 2차 공무원 피드백 반영 및 UI 검증 결과

## 목적

2차 공무원 피드백을 반영해 공통 레이아웃, 좌측/우측 패널, 일정, 업무대화, 내 지식폴더, 문서작성 화면을 정리하고, Windows Codex 세션의 in-app Browser 기반 컴퓨터유즈 방식으로 실제 사용 흐름을 점검했다.

## 반영 결과

- 상단 타이틀 영역에서 세션 상태, 일정연결/독립 상태, 업무엔진 텍스트 상태를 제거하고 우측 상태/도구 영역과 한 줄 구조로 통합했다.
- 화면 배율 표기 좌우에 `-`, `+` 버튼을 추가하고 기존 네이티브 확대/축소 상태 표기와 연동했다.
- 좌측 메뉴를 픽셀 이미지 아이콘 기반으로 바꾸고 `로컬파일/정보검색`을 `파일찾기`로 변경했다.
- 좌측 대화세션 필터인 전체/일정연결/독립 버튼을 제거했다.
- 우측 패널은 현재 컨텍스트, 승인 요청, 최근 실행을 기본 아이콘 탭으로 유지하고, 일정/파일찾기/지식폴더 상황에만 가까운 일정, 미리보기, 덤프 뷰어 아이콘을 활성화한다.
- 우측 패널의 `작업상세` 항목을 제거하고 관련 액션은 컨텍스트, 승인, 로그, 덤프 뷰어로 재배치했다.
- 일정 화면 제목을 `업무일정 캘린더`로 변경하고 중앙의 일정요약/세션연결/선택상태 대형 표기를 제거했다.
- 가까운 일정은 일정 메뉴 선택 시 우측 패널에서만 확인하도록 조정했다.
- 업무대화의 기능 사용법 문의는 로컬 가이드 답변으로 라우팅하고, 답변 파싱은 긴 문단을 더 짧게 나누도록 보완했다.
- 내 지식폴더 설명을 `그래프RAG로 지식관리`로 변경하고 설정/상태, 색인 처리, GraphRAG 검색을 아이콘형 탭으로 정리했다.
- 색인 처리 화면은 색인완료 파일 수와 색인이력 중심으로 정리하고, 색인완료 파일 수를 누르면 별도 추출 결과 화면으로 이동하도록 분리했다.
- 문서작성 화면에서 입력 템플릿 블록을 제거하고, 업무대화 세션/직접 개요/첨부 파일을 출발점으로 4가지 공공문서 형식에 맞춰 구성하는 흐름으로 재정리했다.

## 컴퓨터유즈 확인

- 확인 방식: Codex in-app Browser로 `http://127.0.0.1:5173` 접속 후 실제 메뉴 클릭 및 화면 상태 확인.
- 공통 레이아웃: 상단 제목, 배율 `- / 100% / +`, 새로고침, 우측 패널 토글, 업무 엔진 신호등이 한 줄에 표시됨을 확인했다.
- 좌측 패널: `파일찾기` 명칭과 픽셀 아이콘이 표시되고, 기존 전체/일정연결/독립 필터가 사라진 것을 확인했다.
- 우측 패널: 기본 아이콘 3개와 선택 메뉴별 옵션 아이콘의 활성/비활성 상태가 구분됨을 확인했다.
- 일정: `업무일정 캘린더` 제목, 중앙 요약 제거, 우측 `가까운 일정` 표출 흐름을 확인했다.
- 내 지식폴더: `그래프RAG로 지식관리`, 아이콘형 탭, 색인 처리 화면, 덤프 뷰어 활성화를 확인했다.
- 문서작성: `자료 기반 문서작성 시작`, `공공문서 구성 및 양식 적용`, 4가지 문서 형식, HWPX/HWTX 사용자 양식 입력 흐름을 확인했다.

## 남은 관찰 사항

- 문서작성의 HWPX 고품질 산출은 UI 흐름과 sidecar 단위 테스트는 통과했지만, 실제 기관 양식 파일을 넣은 시각적 서식 품질 검증은 별도 샘플 양식으로 추가 확인하는 것이 좋다.
- 지식폴더 색인 처리 화면은 현재 개발 DB 기준 색인완료 파일이 0개인 상태에서도 별도 추출 결과 화면 진입 구조는 확인했다. 실제 대량 색인 데이터가 있는 환경에서 표/섹션 구조보기 품질을 추가 점검해야 한다.
