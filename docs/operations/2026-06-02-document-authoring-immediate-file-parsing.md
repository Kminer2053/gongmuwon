# 문서작성 첨부 파일 즉시 파싱 반영

작성일: 2026-06-02

## 목적

문서작성 화면 또는 업무대화 문서작성 스킬에서 첨부/연결된 파일이 있을 때, 파일 경로와 사용자가 적은 메모만 전달하지 않고 가능한 범위에서 본문을 즉시 추출해 WorkSessionBrief에 반영한다.

## 적용 범위

- 텍스트류: TXT, MD, Markdown, CSV, JSON, LOG, YAML
- 구조 문서류: PDF, DOCX, XLSX, PPTX, HWP, HWPX

## 처리 방식

1. 문서작성 요청에서 연결 파일과 직접 첨부 파일 경로를 모은다.
2. 텍스트류는 기존처럼 UTF-8 기준으로 짧은 본문 excerpt를 만든다.
3. 구조 문서류는 `parse_document()`를 호출해 KORdoc/HWPX/PDF/DOCX/XLSX/PPTX 파서 체인을 사용한다.
4. 추출된 제목, 메타데이터, 섹션, 문단, 표 일부를 짧은 evidence 문자열로 압축한다.
5. 추출 실패, 미지원 확장자, 과대 파일은 문서작성 자체를 막지 않고 사용자 입력 메모와 경로 기반으로 폴백한다.

## 사용자 경험 정책

- 즉시 파싱은 대기시간을 늘릴 수 있으므로 20MB 이하 파일만 우선 분석한다.
- 작업 시작 전 화면에서 “지원 문서는 즉시 분석되며 큰 파일은 잠시 걸릴 수 있다”는 안내를 표시한다.
- 작업 중에는 “첨부 파일 업로드”, “파일 본문과 대화 맥락 분석” 상태 문구를 보여준다.
- 깊은 재색인/고품질 GraphRAG 처리는 지식폴더 ingestion의 역할로 유지한다.

## 기대 효과

- 방금 첨부한 HWPX/PDF/DOCX 등도 보고서 초안의 근거로 더 잘 반영된다.
- 사용자가 파일별 주요내용/활용목적을 적으면 즉시 파싱 결과와 함께 WorkSessionBrief의 품질이 높아진다.
- 파싱 실패 파일이 있어도 보고서 작성 흐름이 중단되지 않는다.
