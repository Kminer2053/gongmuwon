# 개발 목표 및 제품 비전

작성일: 2026-07-03

## 1. 한 줄 목표

`로컬 AI에이전트 워크플레이스 : 공무원`의 최종 목표는 공공분야 사무업무자를 위한 `로컬 우선 업무 기억 운영체계`를 만드는 것이다.

단순히 로컬 LLM과 채팅하는 앱이 아니라, 사용자의 업무 폴더, 대화 세션, 일정, 파일, 지식, 문서작성, 실행기록을 하나의 업무 맥락으로 연결하는 Windows 업무 시스템을 지향한다.

## 2. 문제 인식

공공기관 사무업무에서는 문서 자체보다 다음 맥락이 더 중요하다.

- 사용자의 소속 기관, 부서, 팀, 직위
- 담당 업무와 반복되는 업무 패턴
- 기관의 조직체계, 업무분장, 위임전결, 문서관리 규정
- 기존 생산문서, 참고자료, 데이터시트, 회의자료
- 보고서 서식, 문체, 결재 관행, 업무상 표현 규칙
- 일정, 대화, 파일, 산출물 사이의 연결 관계

기존 문서 검색은 파일명 또는 본문 키워드 중심으로 동작한다. Gongmu의 목표는 `문서를 검색`하는 수준을 넘어 `업무를 검색`하고, 그 결과를 대화와 문서작성에 재사용하는 것이다.

## 3. 제품 구조 목표

기본 작업 흐름은 다음 구조를 따른다.

```text
지식폴더 -> 업무대화 세션 <- 파일찾기
업무대화 세션 -> 일정 / 문서작성 / 도구 실행 / 실행기록
문서작성 -> Content Base -> DocumentPlan -> HWPX 산출
```

업무대화 세션은 제품의 중심 단위다. 일정, 연결 파일, 지식폴더 검색 결과, 문서작성 요청은 모두 세션 맥락에 연결되어야 한다.

## 4. 지식폴더 2.0 목표

지식폴더는 메모 후보를 저장하는 기능이 아니라, 사용자가 지정한 로컬 업무폴더를 기반으로 업무지식 DB를 구성하는 기능이다.

목표 처리 흐름은 다음과 같다.

```text
지식폴더 등록
-> Folder Recon
-> Identity/Governance Discovery
-> Document Family Detection
-> Document Role Classification
-> Structured Ingestion
-> Chunking / Embedding
-> SQLite Graph Mirror / Chroma Vector Store
-> Work-Aware GraphRAG 검색
```

1차 구현 원칙은 새 Graph DB 도입보다 SQLite graph mirror와 Chroma optional vector backend를 안정화하는 것이다. 규칙 기반 분석을 기본으로 하고, 낮은 신뢰도 항목만 LLM 보조 판단을 사용한다.

### 핵심 분석 대상

- 기관명, 부서명, 팀명, 직위, 담당업무 키워드
- 조직도, 업무분장표, 직제규정, 위임전결규정
- 문서관리, 보안, 개인정보, 계약, 예산 규정
- 초안, 수정본, 최종본, 제출본, PDF 변환본 등 동일 문서군
- 생산문서, 참고자료, 양식, 데이터시트, 협업문서, 의사결정자료

## 5. 파일찾기 목표

파일찾기는 외부 프로그램 의존을 기본값으로 두지 않는다.

- 로컬 파일명/경로 인덱스를 자체 갱신한다.
- 검색 결과를 업무대화 세션의 연결 파일로 저장한다.
- 연결 파일은 이후 대화, 지식 검색, 문서작성 WorkSessionBrief의 근거로 재사용한다.
- Anything/Docufinder 계열 연계는 레거시 또는 선택형 보조 경로로만 취급한다.

## 6. 문서작성 목표

문서작성은 사용자가 입력한 한 문장을 단순 HWPX로 복사하는 기능이 아니다.

목표 처리 흐름은 다음과 같다.

```text
대화세션/바로 작성 선택
-> 파일등록 및 활용 목적 입력
-> 산출보고서 유형 선택
-> 작업지시 입력
-> WorkSessionBrief 생성
-> Content Base 생성
-> DocumentPlan 생성
-> 양식 매핑
-> HWPX 산출
```

문서작성은 대화 세션의 대화내용, 연결 일정, 연결 파일, 첨부 파일 파싱 결과, 지식폴더 RAG 근거를 충분히 수집한 뒤 공공문서 문체와 보고서 유형에 맞게 재구성해야 한다.

지원 산출 유형은 다음을 기준으로 한다.

- 시행문
- 1페이지 보고서
- 풀버전 보고서
- 이메일
- 별도 지정 양식

## 7. 폐쇄망 배포 목표

공공기관 내부망 또는 인터넷 연결이 제한된 PC에서도 초보 사용자가 설치할 수 있어야 한다.

- Windows NSIS 앱 설치파일 제공
- bundled 업무엔진 자동 시작/복구
- Ollama 설치 감지 및 선택 설치
- `gemma4:e2b` 멀티모달 모델 캐시 포함
- `START_INSTALL_GUI.bat` 안내형 설치 모니터 제공
- `RUN_FULL_VALIDATION.bat` 검증자용 설치/검증/evidence 수집 제공
- 설치 실패 시 로그와 evidence를 회수해 개발 저장소에서 검증 가능

## 8. 사용자 경험 목표

- 사용자는 기술 용어인 `sidecar` 대신 `업무엔진`으로 상태를 이해한다.
- UI는 업무대화 세션 중심 3패널 구조를 유지한다.
- 긴 작업은 진행률, 현재 작업, 경과 시간, 로그 보기 경로를 제공한다.
- 업무대화에서는 도구 실행 여부와 이유를 사용자가 이해할 수 있는 말로 설명한다.
- 문서작성 결과는 파일 경로 카드와 파일/폴더 열기 동작으로 바로 확인 가능해야 한다.

## 9. 품질 기준

제품 완성 판단은 기능 존재 여부가 아니라 실제 업무 맥락을 얼마나 잘 이어주는지로 한다.

- 지식폴더 검색은 출처 문서, 문서 역할, 품질 경고, 관계 정보를 함께 제공한다.
- 문서작성은 Content Base보다 품질이 떨어지는 후속 산출물을 만들지 않아야 한다.
- 설치 패키지는 다른 PC에서 앱, 업무엔진, Ollama, 모델, 기본 LLM 설정까지 검증 가능해야 한다.
- 대화, 일정, 파일, 지식, 문서작성은 끊어진 메뉴가 아니라 하나의 업무 흐름으로 연결되어야 한다.
