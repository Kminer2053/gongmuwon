# Gemma 4 E2B 컴퓨터유즈 1턴 실행팩

- 실행 ID: computer-use-batch-02
- 모델 기준: Gemma 4 E2B (gemma4:e2b)
- 시나리오 수: 10
- 총점: 100

## 실행 지시

이 배치 실행팩을 컴퓨터유즈 한 턴의 작업 지시로 사용한다. 배치에 포함된 시나리오만 실제 앱에서 조작하고, 증거와 점수를 결과 템플릿에 기록한다.

## 점수 입력 규칙

- functional 0~4: 기능이 실제로 동작하는가
- ux 0~3: 진행상태, 오류, 다음 행동이 이해 가능한가
- modelQuality 0~2: 경량모델 답변이 구조화, 출처, 보안, 도구 우선 원칙을 지키는가
- evidence 0~1: 스크린샷, 로그, 산출물 경로 등 검증 증거가 남는가

## 시나리오

### LMUX-02-01 Gemma 4 E2B 로컬 기본 프로필 확인

- 카테고리: 모델 설정과 Gemma 4 E2B
- 우선순위: P0
- 경량모델 초점: Gemma 4 E2B 최적화, 경량모델 정책, 연결 테스트

#### 사전조건
- Windows 개발 또는 설치 실행 환경에서 앱을 시작한다.
- 업무엔진이 실행 가능하며 테스트용 더미 업무 데이터만 사용한다.
- 가능하면 로컬 모델은 gemma4:e2b를 사용하고, 외부 모델은 비교 목적으로만 사용한다.

#### 실시 절차
- 1. 앱을 실행하고 업무엔진 상태가 사용자에게 이해 가능한지 확인한다.
- 2. 좌측 메뉴에서 '모델 설정과 Gemma 4 E2B' 관련 화면 또는 업무대화 흐름으로 이동한다.
- 3. 'Gemma 4 E2B 로컬 기본 프로필 확인'에 해당하는 사용자 입력 또는 클릭 동작을 수행한다.
- 4. 중앙 작업화면과 우측 정보패널이 동시에 필요한 정보를 보여주는지 확인한다.
- 5. 작업 완료 후 최근실행 또는 작업진행에서 결과를 추적한다.

#### 기대 결과
- Gemma 4 E2B 또는 선택 모델의 활성 공급자/모델명이 설정값과 일치한다.
- 경량모델 설정에서도 응답시간, reasoning, 내부추론 노출 방지가 확인된다.
- fetch fail 같은 일반 오류만 남기지 않고 원인과 다음 행동이 보인다.
- 작업 결과는 중앙 화면 또는 우측 패널에서 사용자가 이해할 수 있다.
- 최근실행/작업진행에 한글 설명과 성공/실패 상태가 남는다.

#### 컴퓨터유즈 체크포인트
- 화면 전환 또는 버튼 클릭 후 1초 이내 사용자가 다음 상태를 이해할 수 있는가
- 우측 패널 또는 작업진행에서 상세 증거를 열 수 있는가
- 오류가 발생해도 사용자가 같은 흐름을 계속할 수 있는가

#### 결과 기록 슬롯
- status: pass / partial / fail / blocked / skip
- scores: functional, ux, modelQuality, evidence
- evidence: 스크린샷, 로그, 산출물 경로
- notes/blocker: 사용자경험 메모 또는 블로커
