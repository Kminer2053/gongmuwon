### LMUX-01-01 최초 실행 후 업무엔진 정상 연결

- 카테고리: 앱 시작과 업무엔진
- 우선순위: P0
- 경량모델 초점: 콜드스타트, 자동복구, 상태 피드백

#### 사전조건
- Windows 개발 또는 설치 실행 환경에서 앱을 시작한다.
- 업무엔진이 실행 가능하며 테스트용 더미 업무 데이터만 사용한다.
- 가능하면 로컬 모델은 gemma4:e2b를 사용하고, 외부 모델은 비교 목적으로만 사용한다.

#### 실시 절차
- 1. 앱을 실행하고 업무엔진 상태가 사용자에게 이해 가능한지 확인한다.
- 2. 좌측 메뉴에서 '앱 시작과 업무엔진' 관련 화면 또는 업무대화 흐름으로 이동한다.
- 3. '최초 실행 후 업무엔진 정상 연결'에 해당하는 사용자 입력 또는 클릭 동작을 수행한다.
- 4. 중앙 작업화면과 우측 정보패널이 동시에 필요한 정보를 보여주는지 확인한다.
- 5. 작업 완료 후 최근실행 또는 작업진행에서 결과를 추적한다.

#### 기대 결과

- fetch fail 같은 일반 오류만 남기지 않고 원인과 다음 행동이 보인다.
- 작업 결과는 중앙 화면 또는 우측 패널에서 사용자가 이해할 수 있다.
- 최근실행/작업진행에 한글 설명과 성공/실패 상태가 남는다.

#### 컴퓨터유즈 체크포인트
- 화면 전환 또는 버튼 클릭 후 1초 이내 사용자가 다음 상태를 이해할 수 있는가
- 우측 패널 또는 작업진행에서 상세 증거를 열 수 있는가

- 우측 패널 또는 작업진행에서 상세 증거를 열 수 있는가
- 오류가 발생해도 사용자가 같은 흐름을 계속할 수 있는가

#### 결과 기록 슬롯
- status: pass / partial / fail / blocked / skip
- scores: functional, ux, modelQuality, evidence
- evidence: 스크린샷, 로그, 산출물 경로
- notes/blocker: 사용자경험 메모 또는 블로커
