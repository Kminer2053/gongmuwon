# 문서작성 화면과 업무대화 문서작성 흐름 비교

작성일: 2026-06-02

## 목적

문서작성 기능은 `PUB-DOC-HWPX-SKILL`의 핵심 흐름을 Gongmu 구조에 맞게 적용한다.
목표는 단순히 Markdown을 HWPX로 옮기는 것이 아니라, 업무대화·연결파일·직접첨부·작업지시를 먼저 정리한 뒤 보고서 유형과 양식에 맞춰 내용을 재구성하고 HWPX 산출물로 만드는 것이다.

## 공통 백엔드 파이프라인

두 진입점 모두 최종적으로 같은 문서작성 파이프라인을 사용한다.

1. 입력 수집
   - 대화세션, 연결 일정, 연결 파일, 직접 입력 파일, 작업 설명을 수집한다.
   - 지식폴더 GraphRAG 근거는 제목·목적·outline·세션 맥락을 기준으로 함께 탐색된다.
2. 1단계: 보고서 작성대상 정보정리
   - `AppServices._prepare_document_authoring_markdown()`에서 세션/파일/요청 정보를 묶는다.
   - `compile_document_brief()`가 WorkSessionBrief 성격의 구조화 입력을 만든다.
3. 2단계: 목차구조 및 글쓰기 규칙 적용
   - `build_document_plan()`이 보고서 유형에 맞는 개요, 쟁점, 조치안, 근거 구조를 잡는다.
   - public-doc-to-hwpx식 원칙인 두괄식, 개조식, 한 문장 한 핵심, 불필요 표현 정리를 적용한다.
4. 3단계: 지정 양식 채우기
   - `write_public_hwpx_document()`가 내장 기본 서식 또는 사용자 HWPX/HWTX 양식에 내용을 채운다.
   - 최종 산출물은 HWPX 경로와 검토용 Markdown 경로로 반환된다.

## 문서작성 화면에서 시작하는 경우

사용자가 화면에서 명시적으로 입력값을 정리한다.

1. `대화세션에서 작성` 또는 `바로 작성`을 선택한다.
2. 파일등록 단계에서 세션 연결 파일, 직접 입력 경로, 업로드 첨부 파일을 여러 개 등록한다.
3. 파일마다 `주요내용`과 `활용목적`을 별도로 적는다.
4. 산출보고서 유형을 `시행문`, `1페이지 보고서`, `풀버전 보고서`, `이메일`, `별도지정` 중 선택한다.
5. `별도지정`이 필요한 경우 같은 단계에서 사용자 HWPX/HWTX 양식을 업로드하거나 선택한다.
6. 마지막으로 문서 제목과 작업 설명을 입력하고 `작업 시작`을 누른다.

특징:

- 사용자가 파일별 활용 의도를 직접 입력하므로 보고서 brief 품질이 안정적이다.
- 사용자 지정 양식은 산출보고서 선택 단계에만 연결된다.
- 화면 입력은 `/api/documents/generate`로 전달되고, 백엔드는 동일한 brief-plan-HWPX 파이프라인을 실행한다.

## 업무대화에서 문서작성 지시를 하는 경우

사용자는 자연어로 문서작성 요청을 한다.

1. 규칙 라우터가 `문서작성`, `보고서`, `HWPX`, `1페이지 보고서` 등 고확신 표현을 감지한다.
2. 사용자 확인을 거친 뒤 LLM slot extraction이 문서 제목, 보고서 유형, 지시사항, 대상 독자, 기한 등을 정리한다.
3. 현재 업무대화 세션의 대화내용, 연결 일정, 연결 파일, 최신 Reference/지식폴더 근거를 자동으로 수집한다.
4. 수집된 정보가 `_run_document_create_skill()`을 통해 동일한 문서작성 파이프라인으로 들어간다.
5. 산출 결과는 대화 응답에 `파일 열기`, `폴더 열기`, 검토용 Markdown 경로와 함께 표시된다.

특징:

- 사용자가 별도 화면으로 가지 않아도 빠르게 문서 생성을 시작할 수 있다.
- 연결 파일별 `주요내용/활용목적`은 문서작성 화면보다 덜 명시적이므로, 품질이 중요한 경우 문서작성 화면에서 보완 입력하는 것이 더 좋다.
- 자연어 라우팅이므로 일정등록·파일찾기 요청과 섞인 복합 요청에서는 의도 확인 단계가 중요하다.

## 차이점 요약

| 구분 | 문서작성 화면 | 업무대화 문서작성 |
| --- | --- | --- |
| 입력 방식 | 사용자가 단계별로 직접 입력 | 자연어 지시를 라우터와 LLM이 정리 |
| 파일 맥락 | 파일별 주요내용·활용목적을 명시 입력 | 세션 연결 파일과 대화 맥락을 자동 수집 |
| 양식 지정 | 산출보고서 선택 단계에서 사용자 HWPX/HWTX 선택 | 자연어로 유형을 추론, 사용자 지정 양식은 별도 화면 보완 권장 |
| 품질 안정성 | 높음, 사용자가 근거 활용방안을 직접 통제 | 빠르지만 slot extraction 품질에 영향 |
| 최종 파이프라인 | 동일 | 동일 |

## 운영 판단

- 고품질 보고서나 외부 제출용 문서는 문서작성 화면에서 파일별 맥락을 보완해 생성한다.
- 빠른 초안, 회의록 요약, 세션 정리 보고서는 업무대화에서 바로 지시해 생성한다.
- 두 경로는 결과 품질의 출발점이 다를 뿐, 최종 HWPX 생성 로직은 같은 파이프라인을 사용한다.
