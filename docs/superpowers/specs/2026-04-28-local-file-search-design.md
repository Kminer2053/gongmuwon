# 로컬 파일찾기 자체 구현 설계

> 기준일: 2026-04-28
> 상태: 사용자 승인 방향 반영. Anything/Docufinder 의존은 선택 연계로 내리고, Gongmu 자체 내장 파일찾기를 기본 기능으로 만든다.

## 1. 목표

Gongmu는 사용자가 지정한 로컬 업무 폴더와 그 하위 파일을 빠르게 찾아 업무대화 세션에 연결할 수 있어야 한다.

핵심 사용 흐름:

```text
업무대화 세션
  -> 파일 연결
  -> 로컬 파일찾기
  -> 파일명/경로/문서본문 검색
  -> 검색 결과 선택
  -> 현재 세션에 개별 파일 연결
  -> 문서작성에서 세션 연결 파일을 근거자료로 사용
```

`Reference Set`은 기본 화면에서 전면 노출하지 않는다. 필요한 경우 내부 호환 데이터 구조 또는 고급 기능인 `작업자료 묶음으로 저장`으로만 남긴다.

## 2. 외부 Anything 의존성 판단

Anything/Docufinder는 로컬 문서 검색 도구로 좋은 벤치마킹 대상이지만 Gongmu의 필수 런타임 의존성으로 두지 않는다.

이유:

- 라이선스가 Business Source License 1.1이고 프로덕션 사용은 별도 상용 라이선스가 필요하다.
- Gongmu에 필요한 1차 기능은 고급 OCR/시맨틱 검색 전체가 아니라 빠른 파일명/경로/본문 검색이다.
- 현재 sidecar에는 이미 지식폴더 스캔, 파일 메타데이터 저장, 문서 본문 추출, 업무대화 파일 연결 API가 있다.
- 외부 앱 실행, 검색어 붙여넣기, 결과 경로 수동 import는 UX가 끊기며 공공기관용 설치/운영 단순성에도 불리하다.

따라서 Anything은 `외부 고급검색 도구 열기` 선택 기능으로 유지하고, 기본 파일찾기는 Gongmu 자체 구현을 사용한다.

## 3. 참고한 오픈소스/기술 방향

- Recoll: 데스크톱 전체 본문 검색 도구. 파일명과 문서 내용을 모두 검색하고 Windows/Mac/Linux를 지원한다. 구조적으로는 `폴더 인덱싱 -> 문서 추출 -> 검색 인덱스 -> 미리보기/열기` 흐름이 참고 대상이다.
- ripgrep: 설치형 인덱스 없이 매우 빠른 실시간 텍스트 검색을 제공한다. 즉시 검색 fallback이나 개발자용 텍스트 검색 패턴 참고 대상이다.
- SQLite FTS5: SQLite 내장 전문검색 모듈. Gongmu sidecar가 이미 SQLite를 쓰고 있으므로 별도 검색 서버 없이 파일명/경로/제목/본문 발췌 검색 인덱스를 구성하기 좋다.
- Docufinder/Anything: HWPX/PDF/Office/OCR/시맨틱까지 포함한 고급 로컬 검색 제품 방향의 참고 사례다. 코드는 복사하지 않고 기능 요구와 UX만 참고한다.

## 4. 1차 구현 범위

### 포함

- 지식폴더에 등록된 소스 폴더 파일을 파일찾기 화면에서 직접 검색한다.
- 검색 대상은 파일명, 상대경로, 추출 제목, 본문 발췌, 추출 본문이다.
- 검색 결과는 파일 단위 카드로 표시한다.
- 결과 카드에서 `세션에 연결`, `파일 열기`, `폴더 열기`, `경로 복사` 액션을 제공한다.
- 현재 업무대화 세션이 있으면 `세션에 연결`이 기본 액션이 된다.
- 이미 연결된 파일은 `연결됨` 상태로 표시하고 중복 연결을 방지한다.
- Reference Set 생성 폼은 기본 파일찾기 화면에서 제거하거나 `고급: 작업자료 묶음으로 저장`으로 접는다.
- Anything 영역은 `외부 고급검색`으로 축소한다.

### 제외

- 전체 드라이브 MFT/USN Journal 저수준 색인
- OCR 고도화
- 시맨틱/벡터 검색 고도화
- Docufinder 코드 복사 또는 번들
- 자동 백그라운드 파일감시

## 5. 아키텍처

### Sidecar

기존 `knowledge_source_files`를 파일찾기 인덱스의 1차 원천으로 사용한다.

1차에서는 별도 테이블을 만들기보다 `/api/files/search` 또는 기존 `/api/knowledge/search` 확장을 통해 파일 결과를 반환한다. 검색 성능이 부족하면 다음 단계에서 `file_search_index` FTS5 가상 테이블을 추가한다.

검색 랭킹 기준:

1. 파일명/제목 직접 포함
2. 상대경로 직접 포함
3. 토큰 겹침 수
4. 본문 발췌/추출 본문 매칭
5. 최신 수정일

### Desktop

`로컬파일/정보검색` 화면을 `파일찾기` 중심으로 재배치한다.

상단:

- 현재 연결 대상 세션
- 통합 검색창
- 검색 범위 안내

중앙:

- 자체 검색 결과
- 각 결과의 파일명, 경로, 본문 발췌, 상태
- 연결/열기/폴더/복사 액션

하단 또는 접이식:

- 외부 Anything 고급검색
- 고급 작업자료 묶음 저장

## 6. 성공 기준

- 사용자는 Anything 없이도 Gongmu 안에서 파일명으로 파일을 찾을 수 있다.
- 본문 추출된 문서는 검색어가 본문에만 있어도 찾을 수 있다.
- 검색 결과에서 개별 파일을 현재 업무대화 세션에 바로 연결할 수 있다.
- 연결된 파일은 업무대화 화면의 `연결 파일 N개`에서 확인된다.
- Reference Set 개념을 몰라도 파일찾기와 문서작성 흐름을 사용할 수 있다.

## 7. 후속 확장

- SQLite FTS5 기반 인덱스 테이블 도입
- 폴더 변경 감시 및 증분 재색인
- HWP/PDF/OCR 품질 강화
- 검색 결과 노드와 지식그래프 연동
- 문서작성 Content Base가 세션 연결 파일을 직접 입력으로 받는 구조
