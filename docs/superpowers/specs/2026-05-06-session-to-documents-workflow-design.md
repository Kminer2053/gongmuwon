# 업무대화 세션 기반 문서작성 고도화 설계

## 목표

업무대화 세션을 공무 앱의 기본 작업 단위로 두고, 해당 세션에 쌓인 대화내용, 연결 일정, 연결 파일을 문서작성의 출발점으로 자연스럽게 넘긴다. 문서작성 화면에서는 대화세션 기반 작성과 세션 없이 바로 작성하는 흐름을 모두 지원하되, 최종 산출은 `Content Base(Markdown) -> Template -> HWPX` 구조를 유지한다.

## 사용자 흐름

### 1. 업무대화에서 문서작성으로 이어가기

- 업무대화 화면의 현재 세션 액션 영역에 `문서작성으로 이어가기` 버튼을 둔다.
- 버튼을 누르면 `문서작성` 화면으로 이동한다.
- 문서작성 화면은 `대화세션에서 작성` 모드로 전환된다.
- 현재 세션이 자동 선택된다.
- 문서 제목은 세션명을 기반으로 자동 채운다.
- 문서 목적은 "업무대화 세션 기반 정리"로 자동 채운다.
- Content Base 생성 시 아래 정보를 본문에 포함한다.
  - 업무대화 메시지
  - 연결 일정
  - 연결 파일
  - 사용자가 추가 입력한 작성 개요
  - 출력 문서 유형과 작성 슬롯

### 2. 문서작성에서 세션 선택 후 작성

- 문서작성 화면 상단에 작성 출발점을 둔다.
- `대화세션에서 작성`을 선택하면 세션 선택 드롭다운을 보여준다.
- 선택된 세션의 대화 수, 연결 일정, 연결 파일 수를 요약 카드로 보여준다.
- 세션이 없어도 기존 Reference Set 선택은 보조 입력으로 유지한다.

### 3. 바로 작성

- `바로 작성`을 선택하면 대화세션 선택 없이 문서를 만들 수 있다.
- 사용자는 작성 개요를 입력한다.
- 관련 파일 경로는 붙여넣기 방식으로 연결할 수 있다.
- 직접 작성 흐름도 Content Base에 작성 개요와 관련 파일 목록을 남긴다.

## 출력 유형

문서 출력 유형은 HwpxMaker 스킬 흐름에 맞춰 아래 네 가지를 기본으로 한다.

- 시행문
- 1페이지 보고서
- 풀버전 보고서
- 이메일

`자동 선택`도 유지하여 사용자가 유형을 고르지 않으면 제목, 목적, 기존 템플릿 힌트를 바탕으로 자동 선택한다.

## 사용자 양식 파일

- 사용자는 `.hwpx` 또는 `.hwtx` 파일을 업로드할 수 있다.
- 업로드 파일은 `runtime-workspace/documents/templates` 아래에 저장한다.
- 선택된 양식 경로는 Content Base 메타데이터에 저장한다.
- 최종 HWPX 생성 시 사용자 양식 파일을 열 수 있으면 그 파일을 바탕으로 산출물을 만든다.
- 사용자 양식 파일을 열 수 없으면 명확한 오류로 중단한다.

## 작성 슬롯

문서작성 UI와 sidecar API는 아래 작성 슬롯을 받는다.

- 수신/대상
- 예상 분량
- 긴급도
- 추적성 필요 여부
- 공식 서식 필요 여부
- 요청 조치
- 기한
- 보안 수준

이 값들은 Content Base와 HWPX 검토용 Markdown에 남겨 추후 문서작성 스킬 고도화의 입력으로 재사용한다.

## 제외 범위

- 실제 관인/전자결재 서식 완전 자동 완성은 이번 범위에서 제외한다.
- HWPX/HWTX 내부 필드명 자동 매핑은 이번 범위에서 제외하고, 현재는 사용자 양식 위에 문서 내용을 이어 붙이는 방식으로 처리한다.
- 대형 파일 본문 전체 첨부는 이번 범위에서 제외하고, 관련 파일 경로 연결부터 우선 구현한다.
