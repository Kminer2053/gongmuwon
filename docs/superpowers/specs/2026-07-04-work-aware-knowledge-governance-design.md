# Work-Aware 지식관리체계 설계서 (T-01) — 사용자 합의 기반 통합안

- 작성일: 2026-07-04 · 상태: **설계 확정 (구현 대기)** — 라운드 3까지 사용자 합의 완료
- 합의된 결정: 확정 UX = **편집형 트리 + 확인질문** / 적용 수위 = **자동 적용 + 저확신만 큐** / 버전 처리 = **대표 카드 + 버전 이력 접힘** / 프리셋 어휘집 = **공무원 공통 1종으로 시작**
- 원천: 사용자 3단계 설계 철학 + 과거 GraphRAG 파이프라인 구상(2026-05) + 문서작성 "구조 검토 게이트" 패턴 재적용

## 0. 통일 원리

> LLM/분석기는 **초안**을 만들고, 사용자는 **검토·수정·확정**하고, 시스템은 확정본(SCHEMA.md)을 **결정적으로 적용**한다.

지식체계 수립은 문서작성 구조 게이트와 같은 패턴의 재적용이다. "대화형"의 본질은 문답 개수가 아니라 주도권과 확인이다.

## 1. 전체 파이프라인 (사용자 3단계 + 구 GraphRAG 구상 융합)

```
[제품 내장 프리셋]                              ← 1단계: 사전 셋팅
  · 공공 문서역할 8종: 규정/지침 · 업무매뉴얼 · 계획(안) · 보고서 ·
    회의자료 · 공문/시행문 · 양식/서식 · 참고자료 (+임시/백업 = 그림자 유형)
  · 공무원 공통 업무 어휘집(사업계획 수립·성과평가·예산·정기회의·대외협력…)
  · 파일명 규칙 라이브러리: (최종)(안)(붙임) vN 날짜프리픽스(260408) 회차(2차)
  · 폴더 접두사 관행: □주요□/■참고■/□기타□
        ↓
[니즈 파악]                                     ← 2단계: Identity/Governance Discovery
  · 짧은 질문 3~5개: 기관·부서·직무 / 지식관리 목적(개인 기억|인수인계|보고 생산성)
  · 선택적 근거 첨부: 업무분장표·직제규정·조직도 → 부서 공식 업무범위 추출
  · 폴더 스캔 중 governance 문서 자동 탐색(파일명: 업무분장·직제·분장표…)
    → 발견 시 "이 문서로 업무 범위를 파악할까요?" 확인
        ↓
[Folder Recon + 패턴 인식]                      ← 3단계 전반 (기존 스캔 확장)
  · 폴더 구조·명명 관행 분석 (접두사→중요도, 폴더명→업무 후보)
  · Document Family Detection:
    - 동일 해시(구현됨) + 유사 파일명 클러스터((1017)/(1118) 날짜 변형, v1/v2/최종)
    - 가족 내 최신본/공식본/참고본 판정 (수정일 + 파일명 신호)
        ↓
[융합 → 분류체계 초안]                           ← 프리셋 × 니즈 × 실폴더 패턴
  · 업무 트리(허브 후보 + 폴더 매핑) / 유형 매핑 규칙 / 가족 처리 방침
        ↓
[편집형 트리로 확정]                             ← 합의된 UX
  · 이름수정/병합/분리/제외 편집 + 애매한 항목 2~3개만 확인질문
  · [이 체계로 확정] → SCHEMA.md 저장 (사람이 읽고 편집 가능한 "위키 헌법")
        ↓
[일괄 태깅: 자동 + 저확신 큐]                    ← 합의된 수위
  · 결정론 매칭(폴더→업무, 파일명→유형) 자동 적용
  · 저확신·충돌만 "분류 대기 큐"로
        ↓
[Quality Report]                                ← 구 GraphRAG 구상 이식
  · 부족 정보 / 충돌 문서 / 중복 / 최신본 불명확 / 사용자 확인 필요사항
  · 홈 브리핑(D-06) 카드 + 지식폴더 대시보드에 노출
        ↓
[Work-Aware 위키 재구성]
  · work-areas/<업무>.md 허브: 개요 → 핵심 문서(공식본·최신) → 유형별 문서 →
    관련 업무기록(work/) → 연결 일정
  · 문서 카드 front matter: work_area × role × family(최신본 여부) × 연도
  · index.md 업무 중심 재편 / 검색에 업무·유형 필터
```

## 2. 구 GraphRAG 구상의 이식 판정

| 구 단계 | 이식 여부 | 위키 체계에서의 형태 |
| --- | --- | --- |
| Folder Recon | 유지 | 기존 스캔 + 명명 관행 분석 확장 |
| Identity/Governance Discovery | **핵심 이식** | 니즈 파악 단계 + governance 문서 자동 탐색 |
| Document Family Detection | **핵심 이식** | 파일명 클러스터+최신본 판정 → 가족당 카드 1장(이력 포함) |
| Role Classification | 유지 | 문서역할 8종 프리셋 (유형 축) |
| Structured Ingestion | 구현됨 | parse_document + 위키 카드 |
| Ontology Graph Build | **대체** | 그래프 DB 대신 위키 링크(허브↔카드↔업무기록 backlink) |
| Search Index (Vector/Graph/BM25/Meta) | 축소 유지 | FTS5 + 메타 필터(업무·유형·최신본). Vector는 보류 |
| Quality Report | **핵심 이식** | 색인 후 품질 리포트 + 분류 대기 큐 |

## 3. 라운드 3 확정 결과

1. **문서 가족 버전 처리 = 대표 카드 + 버전 이력 접힘** (확정) — 가족당 카드 1장(최신·공식본 대표), 카드 내 "버전 이력" 접힘 목록(파일명·수정일·경로), 검색은 전 버전 히트 가능하되 결과에 "이전 버전" 배지.
2. **프리셋 어휘집 = 공무원 공통 1종으로 시작** (확정) — 사업계획·예산·성과평가·회의운영·보고·감사·대외협력 등 공통 업무 어휘. 기관유형 분화는 니즈 파악 응답이 쌓인 뒤 후속.
3. **니즈 파악 질문 세트 (v1, 구현 시 문구 조정 가능)** — ①기관 유형(중앙/지자체/공공기관·공사) ②부서명과 담당 업무 한 줄 ③지식관리 목적(개인 기억|인수인계 대비|보고서 생산성) ④업무분장표·직제규정 첨부(선택, 건너뛰기 가능) — 4문항 + 선택 첨부 1로 확정. 폴더 내 governance 문서 자동 탐지 시 ④는 역제안으로 대체.
4. **업무 허브 문안 수위 (기본값 채택)** — 결정론 구성(문서 목록·최신본·통계)이 기본, LLM 개요 문장은 "요약 보강" 실행 시에만 추가 (T-02 경량 원칙과 일관).
5. **태깅 확신도 규칙** — 구현 단계에서 상세화: 폴더 직매핑=高, 파일명 패턴 단독=中, 신호 충돌·무신호=低(큐행). 낮은 확신 기준은 운영하며 조정.
