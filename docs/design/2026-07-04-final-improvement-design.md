## 1. 설계 원칙 (전 작업 공통)

1. **Codex 시각 언어 유지** — 라이트 글래스·잉크(#111214)·그린 액센트·radius 12–24. 재설계는 구조와 컨트롤이지 톤이 아니다.
2. **구조 검토 게이트 패턴** — LLM은 초안, 사람이 검토·확정, 시스템이 결정적으로 적용. (문서작성·지식분류·수정지시 모두 동일 패턴)
3. **경량 로컬모델 우선** — LLM 없이 동작하는 결정론 경로가 기본이고, LLM은 보강이다. 입력 토큰은 예산 관리한다(T-02).
4. **내부 용어·원시 값의 사용자 노출 금지** — enum·키·id·경로·영어 라벨은 한글 라벨 매핑을 경유한다. 산출물(HWPX) 본문도 포함.
5. **모든 상태에 다음 행동** — 빈 상태·실패·완료에는 반드시 CTA가 있다. 실패는 입력을 보존하고 제자리 재시도를 제공한다.
6. **후크는 절제** — 자동 패널 열림·토스트·제안 카드는 "사용자 결정이 필요한 순간"으로 한정하고 나머지는 배지로 알린다.

## 3. 영역별 개선 설계

각 영역의 상세 항목·코드 근거는 백로그 ID로 참조한다. **굵은 ID**는 해당 영역의 핵심.

### 3.0 전역 (디자인 시스템·용어·상태 정책)

- **용어 사전 확정 후 전수 치환** (**G-09~G-14**, C-07): 색인(≠인덱싱) · 지식위키(단일 표기) · 작성 콘텐츠(Content Base 제거, '컨텐츠'→'콘텐츠') · 업무대화 세션 · 직접 작성 · 환경설정 · GraphRAG 표기 전면 제거. **산출물 HWPX 본문의 "Content Base" 문구가 최우선**(G-11 — 결재 문서 노출).
- **토스트 정책** (**G-01**, G-02, G-03): pushToast dedupe(동일 메시지 타이머 리셋), silent 폴링 실패는 토스트 금지 → 엔진 미연결 지속 배너(J-11과 세트), success 톤 신설, 원시 예외 노출 금지.
- **버튼·아이콘 마감** (G-04~G-06, C-10): 첨부 label→button(키보드 접근), 파일 열기=file/위치 열기=folder-open 통일, 톱바 icon-button 마이그레이션 완료, dead CSS 제거, 작업/최근실행 전용 아이콘 분리.
- **상태 패턴 규칙화** (G-07, G-08): disabled에는 사유 title 필수, 목록형 빈 상태는 EmptyState 컴포넌트, 로딩 접미 "…" 1종.
- **응답 마크다운 렌더러 재작성** (**F-05, P0**): `*`/`•`/`◦` 블릿 정규화, trim 전 깊이 계산으로 중첩 리스트, 단일 개행 보존, 문장 재분할 제거.
- **출처 표현 구조화** (**F-01**): 프롬프트는 "(출처: 제목)"만, 경로는 응답 카드 하단 출처 칩(열기/복사)으로. 채팅·근거답변 공통.
- **컨텍스트 예산 관리자** (**T-02**): 최근 N턴+증분 세션 요약, 첨부 임시 FTS 색인+질의연동 발췌, "N자 중 M자 반영" 배지, 토큰 추정치 기록.

### 3.1 홈 "오늘의 브리핑" (신설, D-06)

- 카드 5종: 오늘 일정(C-01 수정된 '다가오는 일정' 유틸 공유) / 이어서 하기 / 지식 브리핑 / 오늘의 제안(일정×FTS 결정론 매칭) / 기능 후크.
- **첫 실행 온보딩 모드** (J-06): 데이터 0이면 카드가 ① LLM 연결(→환경설정) ② 지식폴더 등록·색인 ③ 첫 세션 시작 3단계 가이드로 대체.
- 분류 대기 큐(T-01)·승인 대기·알림(F-20) 랜딩 지점.

### 3.2 업무대화

- **첫 실행** (J-01): 빈 상태에 [새 세션 만들기] CTA, 장기적으로 세션 없이 컴포저 노출→첫 전송 시 자동 생성.
- **LLM 미설정 배너** (J-02): 공통 판정 유틸 + [환경설정으로 이동].
- **실패 복구** (J-03, F-02): 실패 시 draft·첨부 복원 + [다시 시도], 실패 배지(적색)·한국어 요약·접힌 상세·[환경설정 이동].
- **첨부** (F-04, G-04): 클립보드 이미지 붙여넣기 + 드래그&드롭 → 기존 업로드 흐름 합류, 트리거 button화.
- **파일 연결 모달** (D-04): 툴바 [파일 연결] → 검색+연결 모달(파일명 인덱스 재사용). 삭제되는 파일찾기 화면의 대체 경로(J-14③).
- 팝오버 일관성(J-12): Escape/바깥클릭 닫기, 연결 파일 "외 N개".
- 플레이스홀더 카피 교체(F-03), provider/model 원시 id는 한글 라벨+툴팁(G-15).

### 3.3 문서작성

- **탭 재정렬** (**F-09**): 작성 콘텐츠(편집·수정지시) → 미리보기(확인 + [이대로 문서 생성]) → 최종(산출물·승인·다운로드 전용). Content Base 안내문 제거(F-11).
- **미리보기** (D-02, M-09): rhwp WASM 종이 미리보기 + 클라이언트 즉시 렌더(편집 반영), 근사 고지 + 텍스트 이중 뷰.
- **LLM 수정 지시** (**F-08**): `POST /authoring/revise` (현재 구조+지시→검증된 새 구조) + 작성 콘텐츠 탭 지시 입력창 + 변경 요약. Ctrl+Enter 실행(J-12).
- **작업 보존** (J-04): 재생성/중단/실패 시 이전 구조 스냅샷 복원.
- **참고자료 카드화** (F-07): 링크 label을 위키 카드 제목 우선으로, 카드형(아이콘+제목+개요) 표시.
- 세션 0개 데드엔드 CTA(J-05), 임의서식(ax 패턴)은 F-10으로 후속 검토.

### 3.4 일정

- **주간 시간축 그리드** (**F-19, P0**): top=시작시각·높이=길이·now line, 이벤트 블록은 "시각+제목", 요일 헤더 단일화, 하단 보기 셀렉트 제거, 카드 반복 안내문 제거.
- **알림 신설** (F-20): remind_before 필드 + 사이드카 스케줄러 + Tauri OS 알림 + 클릭 시 일정/세션 랜딩.
- 확대 내성은 검증 통과(100~150%) — 유지.

### 3.5 내 지식폴더

- **대시보드 = 컨트롤 패널** (F-12): 카드 액션화(재색인/보강 토글·시작), 색인 이력 확충, 중복 바로가기 제거.
- **색인 처리 재배열** (F-13): 폴더+액션 → 진행 상태 → 보강 → 진단(접힘). 위키 구성 기준 설정 = T-01 SCHEMA.md 편집 진입점. failed 잡 [다시 색인](J-10), 덤프 뷰어→"색인 상세 로그" 순화(C-12).
- **검색 = 단일 질의 모델** (F-14): 방법 선택(키워드|근거답변)+Enter, 결과는 최신 1건+히스토리 칩, 상호 리셋. 결과 카드 액션 확장(J-08: 위키 카드·세션 연결·대화 이어가기), 원인 분기 빈 상태(J-07).
- **위키** (F-18, J-09, F-15→T-01): 페이지 유형별 템플릿(메타 헤더·개요 박스·접이식 아웃라인·키워드 칩·관련 카드 그리드), [이 문서로 질문하기]/[관련 검색] 핸드오프, Work-Aware 허브 구조는 T-01 구현과 함께.

### 3.6 실행기록

- **행 리스트 재설계** (F-17): 오늘/어제 그룹, 시간·한국어 요약·결과 배지, 카테고리 필터 칩, 실패 행 강조, raw JSON 접힘. 구 기능 라벨(file_org 등)은 과거 기록 표시용 유지(C-11).

### 3.7 환경설정

- **프로필 카드 중심** (F-16): 저장 프로필 카드 목록 + [이 프로필 사용] 원클릭 + [수정]/[새 프로필], 섹션 = 모델 연결/개인화/시스템 정보(접힘).
- [LLM 연결 테스트]의 무단 저장 제거(J-13: dry-run 또는 정직 표기+복원), 원시 값·영어 라벨 한글화(G-15), anything 문구 제거(J-14).

### 3.8 우측 컨텍스트 패널 (전면 정비)

- **정보 정직성** (C-01~C-03): 가까운 일정 미래 필터 버그 수정, 승인 카드 한국어 라벨+대상 제목(target_label), resource_key 노출 제거.
- **카드 재설계** (C-04, C-06): 죽은 상세 렌더러 정리 → '현재 컨텍스트' 카드(선택 세션/일정 클릭 이동 + 메뉴별 유효 정보), 응답 맥락 상세 세션 전환 리셋.
- **작업 패널** (C-08, C-13, M-08): 활성 잡 상단 고정+카운트 모수 일치, cancel_requested 비활성 표기, 활성 잡 로그 증분 갱신.
- **행동 존중** (C-09, C-05, C-10): 자동 열림은 승인·실패 한정+배지 대체, 승인 후 다음 단계 CTA, 탭 아이콘 분리+aria-pressed+수량 배지.
- 패널 상태 영속+키보드 리사이즈(C-14).

### 3.9 삭제 작업 (D-04·D-05) — 고아 방지 체크리스트

파일찾기·파일정리·도구 제거 시 함께 처리: 미리보기 탭/ContextPanelKey(C-11) · 승인 빈 상태 문구(C-11) · 실행기록 빈 상태 "파일정리"(J-14) · 설정 anything 문구(J-14) · **채팅 [파일 연결]→모달 대체**(J-14③, 필수 선행) · 사이드카 사용법 안내 갱신(G-17) · 죽은 search 분기(C-04).
